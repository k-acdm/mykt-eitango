# 連続日数バグ 原因分析レポート（2026-04-26）

## 事象サマリ

4/26 朝、保護者から「29 日連続だったのが 1 日に戻っている」との問い合わせ。複数の生徒（10 名以上）で `Students.STREAK` が異常な値にリセットされていることが判明。

### 観測パターン

- **完全リセット（STREAK=1）**：22025 笹川夏鈴 / 24018 早川康佑 / 24009 岩倉寿人 / 24027 片山史也（いずれも毎日合格中）
- **部分破損**：23030 髙山遥太 → STREAK=6、24030 松本凌玖 → STREAK=10
- **無傷**：22029 中綾音（35）、24003 古内伶奈（35）、23009 太田杏乃実（34）など

→ 全体障害ではなく **条件依存のバグ**。

## 根本原因

### 1. 主因：`_toDateStr` 正規表現の anchor 不足

[gas/Code.js:43-52](../gas/Code.js#L43)：

```javascript
function _toDateStr(val) {
  if (!val) return '';
  try {
    var s = String(val);
    if (s.match(/^\d{4}-\d{2}-\d{2}/)) return s.slice(0, 10);  // BUG: 末尾 $ なし
    return Utilities.formatDate(new Date(val), 'Asia/Tokyo', 'yyyy-MM-dd');
  } catch(e) { return String(val).slice(0, 10); }
}
```

正規表現 `/^\d{4}-\d{2}-\d{2}/` に末尾アンカー `$` がないため、ISO 8601 形式（例：`"2026-04-25T15:00:00.000Z"`）にもマッチし、先頭 10 文字 `"2026-04-25"` を切り出してしまう。

### 2. 引き金：Day 2 Stage 2（4/24 commit `e6d8398`）の Students キャッシュ

`cache_students_values` キーで Students シート全体をキャッシュ。

- 書き込み：`JSON.stringify(values)`
- 読み出し：`JSON.parse(hit)`

`Date` オブジェクトは JSON シリアライズで **UTC ISO 文字列** に変換される。

```text
LAST_LOGIN セル（Date 型）= JST 2026-04-25 0:00
  ↓ getValues
Date object (UTC 2026-04-24 15:00:00)
  ↓ JSON.stringify
"2026-04-24T15:00:00.000Z"
  ↓ JSON.parse + _toDateStr (バグあり)
"2026-04-24"   ← 本当は "2026-04-25"（JST）。1 日ズレる
```

### 3. 二次要因：`saveAttempt` の setValues 範囲が STREAK 列を含む

[gas/Code.js:619-622](../gas/Code.js#L619)：

```javascript
const preservedStreak = sRows[i][COL_STREAK];  // ← cache 経由（stale 可能性）
sSheet.getRange(i + 1, COL_CLEARED + 1, 1, COL_LAST_TEST - COL_CLEARED + 1)
      .setValues([[newCleared, now, newHP, preservedStreak, today]]);
//                                          ^^^^^^^^^^^^^^ STREAK 列に書き戻し
```

cache が stale だと、`saveAttempt` が **正しい STREAK 値を上書きしてしまう**。

## バグ発動の連鎖シナリオ（22025 笹川の例）

### 4/24（Day 2 Stage 2 デプロイ後）

1. cache miss → loader で Students シートを読む
2. `LAST_LOGIN` セルが Date 型のため Date オブジェクトとして取得
3. `JSON.stringify` で `"2026-04-22T15:00:00.000Z"` に変換、cache に保存
4. cache hit 時、`_toDateStr` が `"2026-04-22"` を返す（本当は JST `"2026-04-23"`）
5. `today = "2026-04-24"`、`missedDays = 2` と誤判定 → `streak = 1`（リセット枝）
6. シートに `STREAK=1`、`LAST_LOGIN="2026-04-24"`（文字列）を書き込み
7. 続く `saveAttempt` が `preservedStreak = 1` を STREAK 列に書き戻し → 永続化

### 4/25 以降

- cache miss が起きるたびに同じ off-by-one が再発
- セル書式が Date のため、設定した `"2026-04-24"` 文字列も Sheets が再度 Date に自動変換
- `_toDateStr` のバグが残り続ける限り、毎日 STREAK=1 にリセットされる

## なぜ生徒によって挙動が違うか

`LAST_LOGIN` セルの **書式が生徒ごとにばらついている**ため：

- **被害生徒**：セル書式が「日付」自動フォーマットされている → JSON roundtrip で off-by-one
- **無傷生徒**：セル書式が「テキスト」または「自動」のまま文字列保持 → JSON roundtrip しても "2026-04-25" のまま
- **部分破損**：cache miss/hit の偶発的タイミングで一部の日のみバグ発動

## 修正方針

### 主因対策（必須）

`_toDateStr` の正規表現に末尾アンカーを追加し、ISO datetime は `Utilities.formatDate` 経由で JST に再フォーマット：

```javascript
function _toDateStr(val) {
  if (!val) return '';
  try {
    var s = String(val);
    // 純粋な日付文字列（_todayJST() が返すフォーマット）
    if (s.match(/^\d{4}-\d{2}-\d{2}$/)) return s;
    // _nowJST() フォーマット（タイムゾーンなしの日時文字列）
    if (s.match(/^\d{4}-\d{2}-\d{2}\s/)) return s.slice(0, 10);
    // Date オブジェクトまたは ISO datetime（タイムゾーン付き） → JST で再フォーマット
    return Utilities.formatDate(new Date(val), 'Asia/Tokyo', 'yyyy-MM-dd');
  } catch(e) { return String(val).slice(0, 10); }
}
```

### Defense in depth（推奨）

- **saveAttempt** の setValues 範囲から STREAK 列を除外
  - 旧: D-H 5 列（CLEARED, UPDATED, HP, **STREAK**, LAST_TEST）
  - 新: D-F 3 列（CLEARED, UPDATED, HP）+ H 単発（LAST_TEST）
- **loginStudent** の setValues 範囲から LAST_TEST 列を除外（同じ stale 書き戻しリスクの排除）
  - 旧: E-I 5 列（UPDATED, HP, STREAK, **LAST_TEST**, LAST_LOGIN）
  - 新: E-G 3 列（UPDATED, HP, STREAK）+ I 単発（LAST_LOGIN）
