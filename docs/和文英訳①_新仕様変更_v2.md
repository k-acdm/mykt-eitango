# 和文英訳① データ仕様変更（新Excelテンプレート対応）

- **作成日**：2026年4月26日
- **対象**：マイ活アプリ「和文英訳①」コンテンツ
- **背景**：4/27（月）からの実運用開始に向け、ふくちさんが Excel テンプレートを再設計。これに合わせてスプレッドシート構造・管理画面UI・GAS流し込み処理を変更する
- **緊急度**：高（4/27 朝までに完成・問題データ投入が必要）

---

## 1. 変更の背景

前スレで `Wabun1Topics` を15列構造（task1〜4、answer1〜4、skip1〜4、word_list ほか）で実装した。

実運用開始直前に、ふくちさんが Excel テンプレート（`和文英訳①_問題データテンプレート.xlsx`）を作成。Excel の構造は：

- 4列構成（曜日 / 種別 / 内容 / 補助列）
- 1週間分が縦展開、月〜日 各日 11〜14行（単語数で変動）
- 各日の構成：問題1 + 日本文 + 問題2 + 問題3 + 問題4 + スキップ + 単語×N + 正解1〜4

この Excel を**そのままコピー（A〜C列のみ）して管理画面に貼り付ければ動く**運用を実現するため、スプレッドシート構造・GAS流し込み処理・管理画面UIを再設計する。

---

## 2. 新仕様

### 2.1 `Wabun1Topics` の新列構成（13列）

| # | 列名 | 型 | 説明 |
|---:|---|---|---|
| A | date | date | 各曜日の日付（yyyy-MM-dd） |
| B | week_no | number | 週番号 |
| C | task1 | string | 問題1の指示文 |
| D | japanese_text | string | 日本文（task1 で並べ替える対象、新仕様で task1 から分離） |
| E | task2 | string | 問題2の指示文 |
| F | task3 | string | 問題3の指示文 |
| G | task4 | string | 問題4の指示文 |
| H | skip_text | string | スキップメッセージ（旧 skip1〜4 を統合） |
| I | word_list | string | 単語（セル内改行で複数語、形式：`日本語 = English`） |
| J | answer1 | string | 正解1（日本文の並べ替え） |
| K | answer2 | string | 正解2（英文） |
| L | answer3 | string | 正解3（否定文 or 疑問文 or その他） |
| M | answer4 | string | 正解4（説明文） |

#### 構造の特徴

- **1日 = 1行**（前スレと同じ思想）
- 1週間分で7行（月〜日）
- 該当しない項目は空欄でOK（例：単語が3つしかない曜日でも、word_list に3行で書く）

### 2.2 旧仕様からの変更点

| 観点 | 旧仕様 | 新仕様 |
|---|---|---|
| 列数 | 15列 | 13列 |
| 日本文の扱い | task1 の指示文 と日本文が同じ列に混在していた可能性あり | **`japanese_text` 列を新設**して分離 |
| skip | skip1 / skip2 / skip3 / skip4 の4列 | **`skip_text` 1列に統合**（番号区別なし） |
| word_list | 同じ形式 | 同じ |
| answer1〜4 | 同じ形式 | 同じ |

### 2.3 既存シートの扱い

- 既存 `Wabun1Topics` シートはテストデータのみで実運用前
- **既存シートの中身を全削除**、新ヘッダー（13列）で作り直す
- データ移行は不要

## 3. 管理画面の入力フォーム

### 3.1 ふくちさんの実運用フロー

1. ふくちさんが Excel テンプレートで翌週分の問題を作成（4列：曜日/種別/内容/補助列）
2. **A〜C列のみ範囲選択してコピー**（D列の補助列は除外）
3. 管理画面の入力フォームに貼り付け
4. 「投入」ボタン → GASがTSVを解析して `Wabun1Topics` に書き込み

### 3.2 入力フォームのUI（既存UIを踏襲）

現状のUIを大きく変える必要はなし。以下の項目を保持：

- 「週の開始日（月曜日）」の日付入力欄
- 「Excelから貼り付け（3列TSV：曜日／種別／内容）」のテキストエリア
- プレビュー領域

### 3.3 入力データの形式

3列TSV：
```
月	問題1	以下の日本文を英文の語順に並び替えなさい。
月	日本文	私たちは毎年冬にこの山の中で大きな熊たちを捕まえます。
月	問題2	1で作った日本文を英語に置き換えなさい。
月	問題3	2で作った文を否定文に変えなさい。
月	問題4	「一般動詞の否定文の作り方」を、『一般動詞の否定文は』で始めて言葉で説明しなさい。
月	スキップ	一般動詞の否定文を塾でまだやってない人は、2と3はやらないで～！
月	単語	毎年冬に = every winter
月	単語	山 = mountain
月	単語	熊 = bear
月	単語	捕まえる = catch
月	正解1	私たちは　捕まえます　大きな熊たちを　この山の中で　毎年冬に
月	正解2	We catch big bears in this mountain every winter.
月	正解3	We don't catch big bears in this mountain every winter.
月	正解4	一般動詞の否定文は、一般動詞の前に don't を置く。
火	問題1	...
（以降、火〜日まで続く）
```

### 3.4 種別の許容パターン

| 種別 | バリエーション | マッピング先 |
|---|---|---|
| 問題1 | `問題1`, `問題１` | task1 |
| 日本文 | `日本文` | japanese_text |
| 問題2 | `問題2`, `問題２` | task2 |
| 問題3 | `問題3`, `問題３` | task3 |
| 問題4 | `問題4`, `問題４` | task4 |
| スキップ | `スキップ`, `スキップ1`, `スキップ2`, `スキップ3`, `スキップ4`（**番号は無視、すべて同じ意味**） | skip_text |
| 単語 | `単語` | word_list（複数行をセル内改行で結合） |
| 正解1 | `正解1`, `正解１` | answer1 |
| 正解2 | `正解2`, `正解２` | answer2 |
| 正解3 | `正解3`, `正解３` | answer3 |
| 正解4 | `正解4`, `正解４` | answer4 |

全角・半角数字の両方を許容する。

### 3.5 投入時の振る舞い

- 同じ週（同じ week_no）のデータが既に存在する場合：**上書き**
- 同じ週のデータがない場合：**新規追加**
- 7日分（月〜日）が揃っていなくてもエラーにせず、揃った日だけ書き込む（部分投入を許容）
- 各曜日の日付は「週の開始日（月曜日）」から自動計算

## 4. GAS 実装変更

### 4.1 影響を受ける関数

| 関数 | 変更内容 |
|---|---|
| `upsertWabun1Topics(weekStartDate, tsvData)` | 新仕様（13列）に対応した解析・書き込み |
| `getWabun1TopicsByDate(date)` | 戻り値構造を新仕様に対応（japanese_text を含む） |
| `submitWabun1` | 採点ロジック内で answer1〜4 を参照する箇所、変更なし（列順は変わるが列名で取得） |
| `_readWabun1TopicsByDate` ヘルパー | 新仕様に対応 |

### 4.2 `upsertWabun1Topics` の処理フロー（新仕様）

```
1. 入力：weekStartDate（月曜の日付）, tsvData（3列TSV文字列）
2. tsvData を行ごとに分割し、各行を [曜日, 種別, 内容] にパース
3. 曜日ごとにグループ化（月/火/水/木/金/土/日）
4. 各曜日について：
   a. 種別を判定して task1 / japanese_text / task2 / task3 / task4 / skip_text / answer1 / answer2 / answer3 / answer4 / word_list に振り分け
   b. 単語は複数行を「\n」で結合して word_list に格納
   c. その曜日の日付を計算（weekStartDate + 曜日オフセット）
   d. week_no を計算（weekStartDate から）
5. 既存の Wabun1Topics シートで、同じ week_no かつ同じ date の行を検索
6. 既存行がある場合：上書き、ない場合：新規追加
7. 7日分すべてを処理
8. 結果（成功/失敗、件数）を返す
```

### 4.3 セルフチェック

投入後、以下を自動チェック：
- 7日分（月〜日）が揃っているか
- 各日の task1, japanese_text, answer1〜4 が空でないか（必須項目）
- 揃っていない場合、警告メッセージで知らせる（エラーで弾かない）

## 5. フロント側の影響

### 5.1 和文英訳①の問題表示画面（index.html）

`getWabun1TopicsByDate(today)` の戻り値が変わるため、フロント側で参照箇所を更新：

**旧**：
```javascript
data.tasks[0].text  // 日本文と指示文が同居していた可能性
```

**新**：
```javascript
data.task1            // 問題1の指示文
data.japanese_text    // 日本文（並べ替え対象）
data.task2            // 問題2の指示文
...
```

- task1 と japanese_text を別々に表示する（指示文と日本文の視認性向上）
- skip_text の表示は1箇所に統一（旧 skip1〜4 を統合）

### 5.2 既存の採点ロジック

`submitWabun1` の採点ロジックは answer1〜4 を参照しているだけなので、列名で取得していれば**変更不要**。
