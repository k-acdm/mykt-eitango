## 1. 概要

### 1.1 目的・位置づけ

「英語長文リスニング＆音読」は、マイ活アプリに追加する新コンテンツ。英検レベル別（4級〜準1級、将来的に1級・準2級プラスを追加予定）の英文に対して、**リスニング → 訳付きリスニング → 正誤問題 → 音読録音送信** の流れで取り組ませる。

**このコンテンツの特徴**

- **「英語音声に慣れる」が主目的**：ネイティブの細部のニュアンスより、英語の音とリズムに触れる時間を確保することを優先
- **音読でアウトプット**：聴いて分かるだけでなく、自分で発音して送信することで定着を促す
- **週次切替**：毎週月曜 3:00〜4:00 に新しい英文セットへ切替。生徒の継続モチベーション維持
- **採点はしない**：正誤問題の⭕❌は表示するが、合格判定なし。送信＝完了でHP付与（学習プロセス自体を肯定する設計）

### 1.2 対象学年・レベル

小学校高学年〜高校生（英検レベルで対象を表現）

| 級 | 想定対象 |
|---|---|
| 英検4級 | 中1〜中2、英検取得を目指す小学校高学年 |
| 英検3級 | 中2〜中3 |
| 英検準2級 | 中3〜高1 |
| 英検2級 | 高1〜高2 |
| 英検準1級 | 高2〜高3 |

将来追加予定：英検1級、英検準2級プラス（準2級と2級の中間レベル）

### 1.3 既存コンテンツとの違い

| コンテンツ | 出題 | 入力 | 採点 |
|---|---|---|---|
| 英単語RUSH | 固定DB | 画面入力 or 書き取り写真 | 完全一致 / Vision OCR |
| 三語短文 | お題日替わり | 画面入力（英文） | AI評価 |
| 和文英訳① | 固定DB | 画面入力（英文） | AI評価 |
| 基礎計算 | 事前生成DB（600問） | 紙に書く → 写真送信 | 厳密照合 |
| **英語長文リスニング＆音読** | **クロ（Claude）が週次生成** | **音声録音送信** | **採点なし、送信＝完了** |

## 2. 基本仕様

### 2.1 5級レベルの構成

#### 現行5級

| 級 | 英文の長さ（語数） | 想定難易度 | HP |
|---|---:|---|---:|
| 英検4級 | 120〜150語 | 易 | 100 |
| 英検3級 | 130〜170語 | 中易 | 100 |
| 英検準2級 | 150〜200語 | 中 | 100 |
| 英検2級 | 180〜230語 | 中難 | 200 |
| 英検準1級 | 200〜250語 | 難 | 200 |

#### 将来追加予定

- 英検1級（250〜300語、HP=200）
- 英検準2級プラス（150〜200語、準2級と2級の中間、HP=100）

### 2.2 実施の流れ

1. 生徒が級を選ぶ
2. 当該級の今週の英文に対して以下を実施：
   - **ステップ1：リスニング**（英文の音声を聴く、テキスト非表示）
   - **ステップ2：訳付きリスニング**（音声を聴きながら日本語訳を読む）
   - **ステップ3：正誤問題3問**（英文の内容に関する正誤判定問題）
   - **ステップ4：解答確認**（⭕❌＋各問の簡潔な解説）
   - **ステップ5：音読録音**（英文を音読して録音）
   - **ステップ6：送信**（録音データを送信）
3. 送信完了でHP獲得（合格判定なし、送信＝完了）

各ステップは戻る・聴き直しが可能。途中離脱した場合は、次回入った時に「途中から再開する」or「最初からやり直す」を選択できる。

### 2.3 獲得HP

#### 級別の獲得HP

| 級 | 獲得HP |
|---:|---:|
| 英検4級 | 100 |
| 英検3級 | 100 |
| 英検準2級 | 100 |
| 英検2級 | 200 |
| 英検準1級 | 200 |

#### 同級複数回・異級累積のルール

- **同じ級を1日に複数回実施しても、HPは初回のみ**
  - 例：英検3級を朝・昼・夜と3回実施 → 初回のみ100HP、2・3回目は0HP（練習モード扱い）
- **異なる級は累積**
  - 例：4級+3級+準2級を実施 → 100+100+100 = 300HP
  - 例：4級+2級+準1級を実施 → 100+200+200 = 500HP
- **1日の上限**：5級フル実施で 100×3 + 200×2 = 700HP
  - ただし他コンテンツとの合計上限ロジック（既存コンテンツ仕様）に従う
  - 基礎計算のような「素点上限」は設けず、級ごとの初回HPがそのまま付く

#### 連続実施ボーナス

既存コンテンツと同じ計算式を適用：

```
獲得HP = 級別HP × Math.ceil(streak / 7)²
```

| 連続日数 | 倍率 |
|---:|---:|
| 1〜7 | 1 |
| 8〜14 | 4 |
| 15〜21 | 9 |
| 22〜28 | 16 |

例：連続10日目で英検準1級を実施 → 200 × 4 = 800HP

## 3. データ設計

### 3.1 スプレッドシート：新規シート構成

#### ① `ListeningPassages`（英文マスタDB）

| 列名 | 型 | 説明 |
|---|---|---|
| passageId | string | 英文ID（`passage_{level}_{YYYYMMDD}`、例：`passage_eiken3_20260427`） |
| level | string | 級（`eiken4`/`eiken3`/`eiken_pre2`/`eiken2`/`eiken_pre1`） |
| weekStartDate | date | この英文が公開される週の月曜日（JST） |
| weekEndDate | date | 公開終了日（次の月曜の前日） |
| status | string | `current`（公開中）/ `archived`（過去分）/ `draft`（投入準備中） |
| passageText | string | 英文本文 |
| passageJa | string | 日本語訳 |
| audioFileId | string | リスニング音声の Google Drive ファイルID |
| audioUrl | string | フロント再生用の音声URL |
| q1 | string | 正誤問題1（英文 or 日本語、§3.2で論点） |
| q1Answer | string | `true` / `false` |
| q1Explanation | string | 正解の解説（簡潔に） |
| q2 | string | 正誤問題2 |
| q2Answer | string | 同上 |
| q2Explanation | string | 同上 |
| q3 | string | 正誤問題3 |
| q3Answer | string | 同上 |
| q3Explanation | string | 同上 |
| createdAt | datetime | 投入日時 |
| createdBy | string | 投入者（ふくちさん運用ならアカウント名） |

#### ② `ListeningSessions`（セッション管理）

| 列名 | 型 | 説明 |
|---|---|---|
| sessionId | string | セッションID（`lst_{studentId}_{timestamp}_{random}`） |
| studentId | string | 生徒ID |
| passageId | string | 取り組んだ英文ID |
| level | string | 級 |
| currentStep | number | 現在のステップ（1〜6、途中離脱からの復帰用） |

| q2Result | string | 同上 |

#### ③ `ListeningHPLog`（獲得HPログ）

**方針**：既存の `HPLog` シートに `source = "listening_{level}"` で書き込む（基礎計算と同じ扱い、`rawHP` カラムも活用）。
→ 新規シート不要。既存ロジックを流用できる。

| passageId | string | 英文ID |

| shareUrl | string | 管理画面から再生するための共有URL |
| submittedAt | datetime | 提出日時 |
| deleteAfter | date | 削除予定日（運用ポリシーは §3.4 で決定） |

### 3.2 英文データの形式

#### 英文本文の形式

- プレーンテキスト（HTMLタグなし）
- 段落区切りは `\n\n`（空行）で表現
- 特殊文字（カーリークォート 'foo' など）は半角クォートに統一
- 級別の語数目安に従う（§2.1）

#### 日本語訳の形式

- 直訳調ではなく、自然な日本語
- 英文の段落と1:1で対応
- 中学英語で習わない構文（仮定法、関係代名詞など）が出る級では、注釈は付けず訳文の中で吸収

#### 正誤問題の形式

- **すべて英語で出題**
- 形式：`The author thinks Japanese food is healthy.` のような肯定文
- 生徒は `True` / `False` のいずれかを選択
- 英文の内容理解を問う問題（リスニングの確認）

**英語出題とする理由**：

- 本コンテンツは「英語音声に慣れる」「英語のアウトプットを促す」が主目的
- 問題まで日本語にすると、英語に触れる時間が分断される
- 英検レベルに合わせた英文での問題は、レベル別の英文難易度と一貫性が取れる
- 4級〜準1級すべてで英語問題、生徒のレベル感に合わせた問題文で出題

#### 解説の形式

- 各問について、**1〜2文程度の簡潔な日本語**
- 英文の該当箇所を示す（「第2段落の冒頭で〜と述べているため」など）
- 英語学習の目的なので、解説は日本語で生徒の理解を助けることを優先

### 3.3 セッション管理

1. **セッション開始**：
   - 生徒が級を選択
   - GAS が `ListeningPassages` から該当級の `status = current` の英文を取得
   - `ListeningSessions` に1行追加（`status = in_progress`、`currentStep = 1`）
   - フロントには英文・訳・問題・音声URLをまとめて返却

2. **ステップ遷移**：
   - 生徒がステップを進めるたびに、フロントが `updateKisoSessionStep(sessionId, step)` を呼ぶ
   - GAS が `ListeningSessions.currentStep` を更新
   - 途中離脱して戻ってきた時は `currentStep` から再開

3. **正誤問題の提出（ステップ3→4）**：
   - 各問の解答（`true`/`false`）を送信
   - GAS が `q1Result`、`q2Result`、`q3Result` を保存
   - 採点なし、各問の正解と解説を返す

4. **音読録音の送信（ステップ5→6）**：
   - 録音データ（webm or mp3、base64エンコード）をGASに送信
   - GAS が Google Drive に保存、`recordingFileId` を `ListeningSessions` に保存
   - `ListeningRecordingsMeta` に1行追加

5. **送信完了処理**：
   - `ListeningSessions.status = completed`、`completedAt` 設定
   - 当日初回なら `hpEarned = 級別HP × 倍率`、2回目以降なら `hpEarned = 0`
   - `HPLog` に1行追加

### 3.4 音読録音データのストレージ

#### 保存先：Google Drive

基礎計算と同じ Google Drive 配下に専用フォルダを用意。

- ルートフォルダ：`マイ活_リスニング音読_録音`
- 年月でサブフォルダを切る：`マイ活_リスニング音読_録音/2026-04/`
- ファイル名規則：`{生徒ID}_{level}_{sessionId}.webm`
  - 例：`S001_eiken3_lst_S001_20260427100000_abc123.webm`

#### 保存期間と自動削除

基礎計算の写真と同じルールを適用：

- 保存期間：**送信日から15日間**
- 削除方法：**GAS の Time-based Trigger** で日次実行
  - 毎日深夜 **3:00〜4:00の間** に起動（基礎計算の写真削除トリガーと同じ時間帯）
  - `ListeningRecordingsMeta` シートを走査し、`deleteAfter <= 今日` の行について：
    1. Google Drive 上の該当ファイルを削除
    2. `ListeningRecordingsMeta` シートの該当行を削除
  - **確認メッセージなし・無条件削除**

#### 保存タイミング

- 音読録音は**毎回保存**（基礎計算の「初回のみ」とは異なる）
- 理由：基礎計算は「字の丁寧さ」を初回確認すれば足りるが、音読は**発音の上達**を継続観察する価値がある

### 3.5 週次切替の仕組み

#### 切替タイミング

- 毎週月曜 3:00〜4:00 の間に GAS の Time-based Trigger で切替実行
- 基礎計算の写真削除トリガーと同じ時間帯

#### 切替処理の内容

1. `ListeningPassages` シートを走査
2. `weekStartDate` が今日（月曜）と一致する `status = draft` の行を探す
3. 該当行を `status = current` に更新
4. 旧 `current` 行を `status = archived` に更新
5. 切替ログを管理シートに記録

#### 投入運用フロー

1. **別スレでクロ（Claude）が週次英文セットを生成**（5級分）
   - 各級について以下の4要素を生成：
     - **1. 英文本体**
     - **2. 日本語訳文**
     - **3. 正誤問題**（英語、3問）
     - **4. 正解と解説**（True/False + 解説日本語、3問分）
2. **TTS事前生成**：英文を音声化、Google Drive にアップロード
   - これは§6で詳細を決定
3. **ふくちさんが管理画面から4枠別々に入力**：
   - 級を選択 → 4つの入力枠が表示される
     - 枠1：英文本体
     - 枠2：日本語訳文
     - 枠3：正誤問題（3問の英文）
     - 枠4：正解と解説（3問分のTrue/False と解説）
   - 入力データは GAS 経由で `ListeningPassages` に書き込まれる（`status = draft`、`weekStartDate` を翌月曜に自動設定）
   - `audioFileId`、`audioUrl` は別途、TTS生成結果のファイルID貼り付け or 手動アップロード
4. **月曜 3:00〜4:00 に自動切替**：上記の切替処理が走る

詳細な管理画面UIは §5 で規定する。

#### 投入忘れ防止

- 管理画面に「次週分の投入状況」を表示
  - 5級すべての `draft` 行が揃っているか
  - 揃っていない場合、警告表示

## 6. 音声・録音の技術仕様

### 6.1 リスニング音声の生成方式（B案：TTS事前生成、確定）

#### 採用するTTSサービス

候補：

- **Google Cloud Text-to-Speech**：日本企業のサービス、Voicesも豊富、月100万文字まで無料枠
- **OpenAI TTS API**：自然な抑揚、コスト中程度
- **Amazon Polly**：歴史長い、安定

**推奨：Google Cloud Text-to-Speech**

- ふくちさんが既に Google Workspace を利用中、APIアクセス管理が一元化できる
- 無料枠で十分（5級×52週=260英文/年、各英文200語平均=52000語/年≪100万文字/月）
- 「Studio」音声（高品質ニューラル）を選べば、機械音声でも違和感少ない

#### 音声品質の保証ライン

「英語音声に慣れる」が主目的のため、**ネイティブ録音レベルではなくとも、教材として違和感なく聴ける品質**を最低保証ラインとする。具体的には：

| 観点 | 保証ライン |
|---|---|
| 発音の正確性 | 音素レベルで正しい（聞き間違いを誘発しない） |
| 抑揚 | 自然なイントネーションパターン（モノトーンではない） |
| 読み上げ速度 | 級別に適切（4級は遅め、準1級は標準速） |
| 音質 | クリアでノイズなし |
| 不自然さ | 固有名詞・略語以外は違和感なし |

Google Cloud TTS の **Studio音声**（en-US-Studio-O / en-US-Studio-Q）または **Neural2音声**（en-US-Neural2-A〜J）を選べば、上記保証ラインは確保できる見込み。初回投入時に必ず1度聴いて品質チェックし、問題があれば声優や速度を調整する。

#### 推奨設定

```
音声：en-US-Studio-O（女性）または en-US-Studio-Q（男性）
読み上げ速度：
  - 英検4級：0.85倍（少しゆっくり）
  - 英検3級：0.90倍
  - 英検準2級：0.95倍
  - 英検2級：1.00倍（標準）
  - 英検準1級：1.00倍（標準）
ピッチ：標準（0.0）
SSML対応：あり（強調・ポーズなどを文章に応じて指定可能）
```

#### 生成プロセス

1. 別スレでクロ（Claude）が英文を生成
2. ふくちさん（or 自動化）が Google Cloud TTS API に投げる
3. 返ってきた mp3 ファイルを Google Drive にアップロード
4. ファイルIDを管理画面の投入フォームに貼り付け

**初期実装での簡素化**：
- TTS生成は手動（ふくちさんがGCP Console から API を叩く、or 別の TTS Web UIサービスを使う）
- 完全自動化は将来拡張

### 6.2 音声ファイルの仕様

| 項目 | 仕様 |
|---|---|
| フォーマット | mp3（汎用性最重視） |
| ビットレート | 96 kbps（音声には十分、ファイルサイズ小） |
| サンプリングレート | 24 kHz |
| チャンネル | モノラル（語学なのでステレオ不要） |
| 想定ファイルサイズ | 約300〜500KB（英文1本あたり） |

### 6.3 音読録音の取得（MediaRecorder API）

#### ブラウザでの録音

```javascript
navigator.mediaDevices.getUserMedia({ audio: true })
  .then(stream => {
    const recorder = new MediaRecorder(stream, {
      mimeType: 'audio/webm;codecs=opus'
    });
    // 録音開始・停止・データ取得処理
  });
```

#### マイク権限

- 初回利用時にブラウザがマイクへのアクセス許可を求める
- 拒否された場合：「マイクへのアクセスが必要です」のエラー表示＋設定方法のリンク
- 許可状態は保存される（次回以降は自動許可）

#### 録音データの形式

| 項目 | 仕様 |
|---|---|
| フォーマット | webm/opus（モバイルブラウザで標準サポート） |
| ビットレート | 48 kbps（音声には十分） |
| サンプリングレート | 16 kHz（モノラル） |
| 想定ファイルサイズ | 約100〜200KB（30秒〜1分30秒の音読） |

#### 容量試算

- 想定：5級 × 20生徒 = 100セッション/週、15日保存
- 平均ファイルサイズ：150KB
- 同時保存最大：100 × 15/7 ≒ 215件 ≒ 32MB
- 基礎計算と合わせても Google Drive 無料枠（15GB）に余裕で収まる

## 7. 週次英文セットの生成・投入運用

### 7.1 別スレでの生成依頼フロー

#### 役割分担

- **本スレ（メインスレ）**：マイ活アプリ全体の仕様・実装・統合
- **別スレ**：英文セットの週次生成専用（運用窓口）

別スレを設ける理由：本スレが実装作業で長くなった時に英文生成依頼が混入すると履歴が複雑化する。生成は週次運用なので独立したスレで扱う。

#### 別スレへの依頼テンプレート

```
今週分（週開始：YYYY/MM/DD 月曜）の英語長文リスニング＆音読セット
を生成してください。5級分（4級・3級・準2級・2級・準1級）。

各級について以下を生成：
1. 英文本体（語数は §2.1 の目安に従う）
2. 日本語訳文（自然な日本語、英文と段落対応）
3. 正誤問題3問（英語、True/False形式）
4. 各問の正解と解説（解説は日本語）

【テーマ】
（指定があれば。例：「日本の伝統文化」「環境問題」「身近な科学」）
（指定がなければ、生徒の興味を引く話題をクロが選択）

【避けるテーマ】
- 政治的・宗教的に偏ったもの
- 過度に暗い・暴力的なもの
- 直近に出題されたテーマ（履歴は別スレで保持）

成果物は4枠別の形式で出力：
[英検4級]
枠1：英文本体
枠2：日本語訳文
枠3：正誤問題（Q1〜Q3）
枠4：正解と解説（Q1〜Q3）

[英検3級] ...
（5級分繰り返し）
```

### 7.2 投入運用のスケジュール

| 曜日 | アクション |
|---|---|
| 木曜 | 別スレに翌週分の生成依頼 |
| 金曜 | クロが生成、ふくちさん確認・微修正 |
| 土曜 | TTS音声生成（GCP TTS API or 手動）、Drive アップロード |
| 日曜 | 管理画面から4枠投入＋音声ファイルID紐付け、`status = draft` で確定 |
| 月曜 3:00〜4:00 | 自動切替実行、`status = current` に |
| 月曜以降 | 生徒が当週分を実施 |

### 7.3 投入忘れ防止

- 管理画面ダッシュボードに「次週分の投入状況」を常時表示
- 5級すべて draft 揃ったら ✅、未投入があれば ⚠️ で警告表示
- ふくちさんの運用ルール：**日曜中に翌月曜以降の分を投入**しておけばOK
  - 月曜 3:00〜4:00 の自動切替トリガーが draft → current に切り替えるため、日曜23:59 までに draft 状態であれば配信される

## 8. HP加算ロジック

### 8.1 級別の獲得HP（§2.3再掲）

| 級 | 獲得HP |
|---:|---:|
| 英検4級 | 100 |
| 英検3級 | 100 |
| 英検準2級 | 100 |
| 英検2級 | 200 |
| 英検準1級 | 200 |

### 8.2 同級複数回・異級累積のルール

- **同じ級を1日に複数回**：初回のみHP付与、2回目以降は0HP（練習モード）
- **異なる級**：累積（4級+3級+準2級+2級+準1級 = 700HP/日）

### 8.3 連続実施ボーナス

```
獲得HP = 級別HP × Math.ceil(streak / 7)²
```

例：

- 連続10日目で英検準1級を実施 → 200 × 4 = 800HP

## 9. 採点ロジック

### 9.1 正誤問題の正解判定（採点というより記録）

- 各問について、生徒の解答（True/False）と正解を比較し ⭕❌ を表示
- **合格判定なし**：何問正解しても何問不正解でも、HP付与に影響しない
- 統計目的でセッションに記録（将来的に分析する可能性あり）

### 9.2 音読録音は採点なし（送信＝完了）

- 音読録音には**いかなる採点も行わない**
- 送信ボタン押下＝完了とみなしHP付与
- ふくちさんが管理画面から音声を聞いて、必要に応じてLINEなどで個別フィードバック（仕組みとしては仕様外）

### 9.3 設計思想

- 「英語音声に慣れる」「英語のアウトプットを促す」ことが主目的
- 生徒に「失敗のプレッシャー」を与えず、毎週続けることを優先
- 結果（合否）より行動（実施）を肯定する設計

## 11. 残課題・今後の拡張

### 11.1 確認事項の状況

すべての設計判断は確定済み：

- §3.2 正誤問題は英語で出題
- §3.4 音読録音 15日保存・3:00〜4:00 自動削除
- §3.1 シート構成（4シート）
- §3.5 管理画面から4枠別入力
- §6.1 TTS は Google Cloud Text-to-Speech 推奨

### 11.2 本実装で対応しないもの（意図的に見送り）

- TTS音声生成の完全自動化（GCP API を直接GAS から叩く）→ 当面は手動アップロード
- 音読録音のAI採点・スコアリング → 仕様書 §9.2 で明示的に「採点なし」
- 過去週の英文の編集 → archived は読み取り専用
- 英文の言語切替（多言語対応）→ 英語のみ

### 11.3 将来拡張候補

#### 短期（リリース後1〜3ヶ月）

- **音読録音の自動文字起こし**：Google Speech-to-Text で文字起こし → ふくちさんの確認工数削減
- **当週の実施統計**：管理画面に「今週何人が何級を実施したか」のサマリ
- **英検1級・準2級プラスの追加**：仕様 §1.2 で予告済み

#### 中期（半年〜1年）

- **TTS自動化**：英文投入時に自動でTTS生成→Drive 保存→ファイルID自動紐付け
- **発音評価**：音読録音とTTS音声を比較して発音スコアを出す（実装難度高）
- **生徒別の苦手傾向分析**：正誤問題の正答率を統計

#### 長期（1年以降）

- **アバター機能との連携**：リスニング合格でHP獲得 → アバター着せ替え
- **英文ライブラリ化**：過去の英文を「練習用」として全級にアクセス可能にする

### 11.4 運用上の注意

#### TTS音声の品質管理

- 生成後は必ず1度聴いて品質チェック
- 不自然な発音があれば手動で再生成
- Studio 音声を使えば大半は問題ないはずだが、固有名詞・略語は要注意

#### 著作権配慮

- クロ（Claude）が生成する英文はオリジナル
- 既存教材・記事の転載は厳禁（仕様書 §7.1 の依頼テンプレートで明示）
- 引用が必要な場合は出典明記
