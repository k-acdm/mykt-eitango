## 🛡️ v13 で確立した重要な設計原則

### 案 A（正解候補ヒント付き判定タスク化）

- カンジー OCR で導入、AI に「自由 OCR」させずに「N 個の候補との一致判定」にする手法
- 構造的に飛躍誤認識を排除できる
- 他コンテンツ（基礎計算 / RUSH5級 / 和文英訳①）への横展開は **v14 以降で実害観察してから判断**

### \_logHP 堅牢化パターン

- 「**HPLog 書き込み先、Students 後**」の書き込み順序を全 HP 付与経路で徹底
- _logHP に戻り値を持たせ、失敗時は早期 return
- 既存データの整合性を保つ（過去欠落は救済しない、今後の発生を止める）
- 全 8 関数で適用済：loginStudent / saveAttempt / submitSango / submitWabun1 / submitKisoAnswer / submitLison / submitKanjiKaki / submitKobunSet

### KanjiSubmissions（軽量版 F'）

- 答案画像は保存しない、判定結果のテキストだけシートに残す
- 容量負荷ゼロで効果測定とトラブルシュートが可能になる
- 同じ思想で SangoSubmissions も新 6 列拡張（ai_category / ai_feedback / ai_reasoning / starred / starred_at / published_in_week）

## 🔄 v13 で確立した運用ルール

### Claude Code 起動時のルール

1. **PowerShell から起動 or タスクバー起動（プロジェクト名 / ブランチ確認）**
2. **必ず dev ブランチで作業**
3. **終了時は `/exit` または Ctrl+C 2 回**（× ボタン禁止、v13 で 1 度事故あり）
4. **大規模変更前に事前確認を出させる**（v13 でサンゴタン実装時に良い習慣として確立）

### Claude Code 完了報告のルール

- 各コミットの SHA を必ず提示（嘘事案防止）
- バックエンド/フロント等の大規模実装は段階的な完了報告（2 段階構成が v13 で確立）

### 本番反映 4 ステップ（CLAUDE.md ルール踏襲）

1. `git checkout main && git merge dev && git push origin main`
2. `cd gas && clasp push`
3. Apps Script エディタで F5 リロード
4. **「デプロイを管理 → 既存デプロイの鉛筆 → 新バージョン → デプロイ」**（「新しいデプロイ」ボタンは絶対 NG）

### worktree 掃除のルール

- Claude Code 起動中は worktree を消さない（v13 で 2 度発生）
- 別セッションを始める前に「`/exit` → worktree 残骸チェック → 削除 → 起動」
