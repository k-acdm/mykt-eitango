# 基礎計算 問題生成 設計原則

このドキュメントは、`scripts/generate_kiso_questions/` 配下の **全級・全フェーズ・全コミッターが必ず守るべき原則** を定義する。

新しい級を追加する時、既存の級を修正する時、Phase 3 以降に拡張する時は、**必ず最初にこのドキュメントを読むこと**。

---

## 原則 1：問題式の分数は常に既約形でなければならない（Phase 1 で確立）

### ルール

問題式に登場する **すべての分数** は、必ず GCD(分子, 分母) == 1 でなければならない。

- ✅ OK：`5/6 - 2/3`、`3/8 + 1/4`、`7/9 + 2/11`
- ❌ NG：`4/6 - 3/6`（4/6 は GCD=2）、`6/9 + 2/9`（6/9 は GCD=3）

### 理由

- 紙教材（春日部アカデミー「計算級別トレーニング」）の問題式は常に既約形で書かれている（教材として当然の仕様）
- 非既約な分数を問題式に出すと、生徒が「先に約分してから計算する」ような不自然な解き方を強いられる
- 仕様書 §6.8 決定 2「解答は既約のみ正解」と一貫させるためにも、問題側も既約に統一すべき

### 適用範囲

問題式に分数が登場する **全ての級**：

- 16級（分数加減）、15級（分数乗除）、14級（分数四則混合）
- 11級（正負の四則混合・分数係数）
- 8級（一次方程式の分数係数）、7級（式の計算・分数係数）
- 6級（連立方程式・分数係数）、5級（式の計算中3・分数係数）
- 2級（平方根の有理化結果）、1級（二次方程式の解が分数の場合）

20級など、原則として分数が出ない級でも、防御的に検証ヘルパーをセルフチェックに組み込む（no-op として動作）。

### 実装上の二段構え

1. **生成側**：`common.sympy_helpers.pick_coprime_numerator()` で互いに素な分子を選ぶ
2. **検証側**：`common.sympy_helpers.assert_problem_fractions_in_lowest_terms()` を各級の `self_check` から呼ぶ

なお **答え側** の既約性・簡約性は別系統の責務（仕様書 §6.8 決定 2/3）で、`common.answer_variants.canonical_for_rational()` が保証する。

### 関連実装

- 発覚事例：Phase 1 実装中の 16級 Band A で `4/6 - 3/6` のような問題式が生成された
- 修正コミット：`a4b743e`（実装側）、`ad4945f`（仕様書側 v1.1）

## 原則 2：Band A〜C は教育的入門として、紙教材より易しめに調整する（Phase 2 グループ②a で確立）

### 趣旨

- Band A〜C は、生徒が当該級に「**初めて触れる**」段階を想定する
- 入門段階で躓くと学習意欲を大きく損なうため、**最も優しい問題から導入する**
- 紙教材の Band A〜C より易しい場合がある（紙教材は印刷物のため難易度勾配が粗い）
- 特に「初回挑戦の HP 獲得体験」を成功させることが本コンテンツの教育設計上の最重要事項

### 具体例（Phase 2 グループ②a での発覚事例）

| 級 | Band | 当初の挙動（教育的に問題） | 調整後 |
|---|---|---|---|
| 8級 | A（ax = b） | `6x = -16 → x = -8/3` のような分数解が出る | a が b の約数を強制し、整数解のみ |
| 8級 | C（比例式） | `13 : 18 = 6 : x → x = 108/13` のような複雑分数解が出る | x が整数 or シンプルな分数（分母 ≤ 4 程度）になる組のみ採用 |
| 6級 | A（連立方程式） | 係数 ±4 を含む `-2x + 4y = -8` のような中程度問題が出る | 係数を ±[1..3] に縮小し、シンプルな整数解 |

### ❗️ 絶対遵守の二次原則

> **「Band A〜C で省いた難易度の問題は、Phase 3（D〜H 拡張）で必ず復活させる」**

- 紙教材と同等の難易度カバレッジを **Band A〜H 全体** で確保するのが最終目標
- 省いた問題タイプは、各級スクリプトに `# TODO_PHASE3:` コメントで明示する
- Phase 3 着手時に [CLAUDE.md](../../CLAUDE.md) と本ドキュメント §「原則 3」を参照して **復活漏れを防ぐ**

### 適用範囲

このドキュメント作成時点では Phase 2 グループ②a までで発見された 8級・6級が対象だが、Phase 2 グループ②b（残りの 5/4/3/2/1 と 10）以降の実装でも、Band A〜C には同じ「入門優先」の判断基準を適用する。

## 原則 3：Phase 3 で必ず復活させるべき「省略済み難易度」一覧

| 級 | 省略した難易度 | 発見 Phase | 復活先（暫定） |
|---|---|---|---|
| 8級 | `ax = b` で解が分数（割り切れない係数） | Phase 2 グループ②a | 8級 Band D 以降 |
| 8級 | 比例式で解が複雑な分数（22/9、108/13 など） | Phase 2 グループ②a | 8級 Band E 以降 |
| 6級 | 連立方程式で係数 ±4〜±6 の中程度（`-2x + 4y = -8` など） | Phase 2 グループ②a | 6級 Band B/C 終盤 or D 以降 |

（Phase 3 着手前、および Phase 2 グループ②b 完了時に追記すること）

## 今後の原則追加について

新しい原則が発見された場合（Phase 3 以降の実装中に発覚するなど）、このドキュメントに追記し、コミットして全コミッターに周知する。

追加時は：

1. 本ドキュメントに節を追加（原則 N として連番）
2. 関連する rank_*.py を修正
3. コミットメッセージで原則番号を明示（例：`fix(基礎計算): 原則 N 違反の修正`）
4. 関連する仕様書 `docs/基礎計算_仕様書.md` への追記が必要なら別コミットで実施
